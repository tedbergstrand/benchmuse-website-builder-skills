# Luthier Profile Schema

This document defines the canonical structure for `luthier-profile.json`. All skills that read or write this file must conform to this schema. The goal is to eliminate ambiguity so downstream skills never have to guess at key paths.

## Top-Level Structure

```json
{
  "identity": { },
  "services": { },
  "policies": { },
  "design_tokens": { },
  "design_brief": { },
  "testimonials": [ ],
  "site_architecture": { },
  "metadata": { }
}
```

## Field Definitions

### `identity` (required)
| Field | Type | Required | Description |
|---|---|---|---|
| `businessName` | string | yes | Legal or trade name |
| `ownerName` | string | yes | Primary luthier's name |
| `location` | string | yes | City, State (never street unless explicitly public commercial address) |
| `profileType` | string | yes | `"builder"`, `"repair"`, or `"hybrid"` |
| `specialty` | string | yes | Core focus area |
| `story` | string | yes | Background narrative for about page |
| `phone` | string | no | Business phone number (only if luthier consents to publishing it) |
| `phoneCTA` | boolean | no | Whether to show a prominent tap-to-call button on the site. Defaults to `true` if `phone` is present. Set `false` if luthier wants to route all contact through a form. |
| `socials` | object | no | `{ instagram, facebook, bluesky, website }` |
| `affiliations` | string[] | no | Professional memberships |
| `authorizedDealerFor` | string[] | no | Brand partnerships / certifications |

### `services` (required)
Keyed by service category. Each category is an object with flexible pricing fields.

**Repair tech example:**
```json
"services": {
  "setups": {
    "name": "Guitar Setup",
    "description": "Full setup including action, intonation, and truss rod adjustment",
    "standard": "$95 and up",
    "floyd_rose": "$135 and up"
  },
  "fretwork": {
    "level_crown_polish": "$200",
    "refret_unbound_rosewood": "$500"
  }
}
```

**Builder example:**
```json
"services": {
  "custom_builds": {
    "name": "Custom Acoustic Guitar",
    "description": "Handbuilt acoustic guitars to your specifications",
    "starting_price": "$3,500",
    "typical_lead_time": "6-9 months"
  }
}
```

### `policies` (required)
| Field | Type | Description |
|---|---|---|
| `appointments` | string | How to book |
| `dropIn` | string | Walk-in policy |
| `hours` | string | Operating hours |
| `payment` | string | Payment expectations |
| `warranty` | string | Guarantee/warranty policy |
| `leadTime` | string | Typical turnaround or build time |
| `communication` | string | Preferred contact method |

### `design_tokens` (required — generated by VVT)
A convenience shorthand derived from `design_brief.color_system` and `design_brief.typography`. Used for quick CSS custom property reference. The `design_brief` is the canonical source — if they conflict, `design_brief` wins.

```json
"design_tokens": {
  "--bg-color": "#111417",
  "--text-color": "#F2F0E9",
  "--primary-color": "#C9A84C",
  "--secondary-color": "#37596B",
  "--accent-color": "#37596B",
  "--font-heading": "'Cormorant Garamond', serif",
  "--font-body": "'Plus Jakarta Sans', sans-serif"
}
```

### `design_brief` (required — generated by VVT)
The complete design specification. This field **must be non-empty** before the Site Builder will start.

| Field | Type | Description |
|---|---|---|
| `stack_recommendation` | string | `"vanilla-html-css"` or `"react-vite-tailwind"` |
| `rationale` | string | Why this stack fits this luthier |
| `site_structure` | string | `"single-page-scroll"` or `"multi-page"` |
| `page_map` | string[] | Pages to build (e.g., `["home", "services", "portfolio", "about", "contact"]`) |
| `hero_concept` | string | Specific hero section description (layout, animation, image treatment, headline) |
| `nav_concept` | string | Navigation pattern description |
| `color_system` | object | See below |
| `typography` | object | See below |
| `signature_interactions` | array | See below |
| `section_patterns` | array | See below |
| `photo_treatment` | string | How photos should be styled (borders, overlays, filters) |
| `texture` | string | Background textures/overlays description |
| `anti_patterns` | string[] | Design choices explicitly avoided |
| `global_effects` | string[] | Site-wide effects (noise overlay, custom selection color, smooth scroll) |

**`color_system` fields:**
| Field | Type | Description |
|---|---|---|
| `strategy` | string | Brief rationale for the palette |
| `background` | string | Hex + description (e.g., `"#111417 — deep slate"`) |
| `surface` | string | Card/panel background color |
| `primary` | string | Primary brand color |
| `accent` | string | Accent/highlight color |
| `text_light` | string | Light text color (for dark backgrounds) |
| `text_dark` | string | Dark text color (for light backgrounds) |

**`typography` fields:**
Each of `heading`, `body`, and `mono` contains:
| Field | Type | Description |
|---|---|---|
| `family` | string | Font family name |
| `provider` | string | Font source (e.g., `"Google Fonts"`) |
| `rationale` | string | Why this font was chosen |
| `usage` | string | Where this font is used |

**`signature_interactions` array items:**
| Field | Type | Description |
|---|---|---|
| `name` | string | Interaction name |
| `description` | string | Specific description of the interaction |
| `location` | string | Which section it belongs to |

**`section_patterns` array items:**
| Field | Type | Description |
|---|---|---|
| `name` | string | Section name |
| `layout` | string | Specific layout description |
| `animation` | string | Specific animation description |

### `testimonials` (optional)
```json
"testimonials": [
  { "author": "Eric O.", "text": "Best setup I've ever had on my Les Paul." }
]
```

### `site_architecture` (optional — preliminary input from Interviewer)
The Interviewer's early recommendation based on scope. Used by VVT as a starting point when setting `design_brief.site_structure` and `design_brief.page_map`. Not used directly by the Site Builder.

**`design_brief` is the canonical source for page structure.** If `site_architecture` and `design_brief` conflict, `design_brief` wins.

```json
"site_architecture": {
  "page_map": ["home", "services", "portfolio", "about", "contact"],
  "site_structure": "multi-page"
}
```

### `metadata` (required)
| Field | Type | Description |
|---|---|---|
| `data_sources` | string[] | How the profile was populated (e.g., "luthier interview") |
| `created_date` | string | ISO date of profile creation |
| `stale_data_flags` | string[] | Warnings about unverified data |
